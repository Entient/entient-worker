# ENTIENT Remote Worker

Turn any idle machine into an ENTIENT operator synthesis + mining + training node.

## What You Need

- **Python 3.10+** — [python.org](https://python.org) (check "Add Python to PATH" during install)
- **Git** — [git-scm.com](https://git-scm.com)
- **Network access** to the coordinator machine (same LAN, or Tailscale for remote)

## Quick Start

### 1. Install (one time)

**Windows:** double-click `install.bat`  
**Linux/Mac:** `bash install.sh`

The installer will:
- Create a Python virtual environment
- Install packages: requests, pynacl, cryptography, cbor2, pyyaml
- Clone 3 repos: entient, entient-agents, entient-interceptor
- Install them in the right order
- Create data directories at `~/.entient/`
- Show you what this machine can run

It will ask for:
- **Coordinator URL** — the IP:port of the machine running `coordinator.py` (e.g. `http://192.168.1.100:8420`)
- **Worker name** — a label for this machine (e.g. `gaming-pc`)

### 2. Bootstrap data (one time)

```
start.bat --bootstrap        # Windows
./start.sh --bootstrap       # Linux
```

This downloads database files from the coordinator so the worker can run more job types. Downloads:
- `outcome_ledger.db` (~90 MB) — unlocks COVERAGE and RETRAIN
- `genome_registry.db` (~28 MB) — unlocks CROSSINDEX (with shapes.db)
- `forwards.jsonl` (~1 MB) — forward history

**shapes.db (~5 GB) is too large to download over HTTP.** Copy it manually via USB drive to `~/.entient/v2/shapes.db` to unlock CROSSINDEX.

### 3. Check capabilities

```
start.bat --check            # Windows
./start.sh --check           # Linux
```

Shows what this machine can actually run and what's missing:

```
  CAPABILITY DETECTION
  ========================================
  Available: compile, coverage, crossindex, mine, retrain

  DATA FILES:
    outcome_ledger.db: 89.6 MB
    shapes.db: 5204.3 MB
    genome_registry.db: 27.9 MB
    forwards.jsonl: 0.7 MB
    bank operators: 8149 files
```

### 4. Start working

```
start.bat                    # Windows
./start.sh                   # Linux
```

The worker will:
1. Auto-detect what it can run
2. Connect to the coordinator
3. Poll for jobs every 10 seconds
4. Execute jobs and upload results
5. Keep running until you press Ctrl+C

## Job Types

| Job | What it does | What it needs |
|-----|-------------|---------------|
| **compile** | Generate operator .py files from JSON specs | Nothing extra (always available) |
| **coverage** | Measure recognition coverage metrics | outcome_ledger.db |
| **crossindex** | Match genomes to shapes for witness discovery | shapes.db + genome_registry.db |
| **mine** | Extract structural patterns from GitHub repos | entient-agent repo installed |
| **retrain** | Retrain the logistic router from outcome data | entient-interceptor + outcome_ledger.db + bank operators |

## Coordinator Setup (on the main machine)

The coordinator must be running before workers can connect:

```
cd entient-agent
python coordinator.py              # starts on port 8420
```

To create jobs for workers:
```bash
# Compile operators from a spec
curl -X POST http://localhost:8420/jobs/create \
  -H "Content-Type: application/json" \
  -d '{"job_type":"compile","spec":[{"name":"my_op","keywords":["test"],"template":"print(1)"}]}'

# Retrain the router
curl -X POST http://localhost:8420/jobs/create \
  -d '{"job_type":"retrain","spec":{}}'

# Check status
curl http://localhost:8420/status
```

Dashboard: `http://coordinator-ip:8420/status`

## Files

| File | Purpose |
|------|---------|
| `worker.py` | Main worker loop — register, heartbeat, claim jobs, execute, upload results |
| `job_handlers.py` | Execution logic for each job type + capability auto-detection |
| `install.bat` / `install.sh` | One-click setup for Windows / Linux |
| `start.bat` / `start.sh` | One-click start (passes args to worker.py) |
| `config.json` | Generated by installer — coordinator URL, worker name |

## Troubleshooting

**"Cannot reach coordinator"** — Check that `coordinator.py` is running on the main machine and the URL/port is correct. Make sure the firewall allows port 8420.

**"entient_agent import failed"** — Run `install.bat` again, or manually: `.venv\Scripts\pip install -e repos/entient-agents`

**"outcome_ledger.db not found"** — Run `start.bat --bootstrap` to download it from the coordinator.

**"shapes.db not found"** — Too large for HTTP download. Copy it via USB from `~/.entient/v2/shapes.db` on the main machine.

**Worker shows only "compile" capability** — Run `--bootstrap` first, then `--check` to see what unlocked.
